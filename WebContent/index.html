Resources, allowed HTTP methods and common returned codes

RETURNED HTTP CODES

  • 200 --> after a delete or after a get when all it's cprrect
  • 201 --> when a new resource is correctly created
  • 404 --> when the resource that you are looking for doesn't exists
  • 503 --> when it is required a connection to Neo4JXML and there is an error retrieving response from Neo4JXML
RESOURCES (common path "http://localhost:8080/rest/resource" is omitted):

/nffg

  • POST - Insert 1 nffg
    ("201 - Created" / "503 - Service Unavailable")

/nffgs

  • GET - Get all nffgs
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")
  • POST - Insert a specified group of nffgs
    ("201 - Created" / "503 - Service Unavailable")
  • DELETE - Delete a specified group of nffgs
    ("200 - OK" / "404 - Not Found")

/nffg/{nffg_id}

  • GET - Get 1 specified nffg
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")
  • DELETE - Delete 1 specified nffg
    ("200 - OK" / "404 - Not Found")

/nffg/{nffg_id}/nodes

  • GET - Get all nodes of a specified nffg
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/nffg/{nffg_id}/node/{node_id}

  • GET - Get a specified Node of a specidief nffg
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/nffg/{nffg_id}/node/{node_id}/links

  • GET - Get all links of a specified node of a specified nffg (all links have as sourceNode the specified node)
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/nffg/{nffg_id}/node/{node_id}/link/{link_id}

  • GET - Get a link of a specified node of a specified nffg (The link has as sourceNode the specified node)
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/nffg/{nffg_id}/links

  • GET - Get all links of a specified nffg
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/nffg/{nffg_id}/link/{link_id}

  • GET - Get a specific link of a specified nffg
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/nffg/{nffg_id}/policies

  • GET - Get all policies of a specified nffg
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")
  • POST - Insert a specified group of policies into a specified nffg
    ("201 - Created" / "503 - Service Unavailable")
  • DELETE - Delete a specified group of policies into a specific nffg
    ("200 - OK" / "404 - Not Found")
  • PUT - Update a specified group of policies into a specific nffg
    ("200 - OK" / "404 - Not Found")

/nffg/{nffg_id}/policy/{policy_id}

  • GET - Get 1 policy of a specified nffg
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")
  • DELETE - Delete 1 specified policy of a specified nffg
    ("200 - OK" / "404 - Not Found")
  • PUT - Update 1 policy of a specified nfg
    ("200 - OK" / "404 - Not Found")

/nffg/{nffg_id}/policy

  • POST - Insert 1 policy into a specified nffg
    ("201 - Created" / "503 - Service Unavailable")

/policies

  • GET - Get all policies of the all nffgs
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/policy/{policy_id}

  • GET - Get 1 policy of the all nffgs
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/verifyPolicy/{policy_id}

  • GET - Verify 1 specified policy getting the resoult
    ("200 - OK" / "404 - Not Found" / "503 - Service Unavailable")

/verifyPolicies

  • POST - Verify a group of policy pasing a list of policies (uset to specify the list of policies IDs) getting all the resoults for all policies
    ("200 - OK" / "503 - Service Unavailable")
